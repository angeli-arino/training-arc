# Accessibility Checklist

Training Arc can be cute and accessible at the same time.

## Keyboard and interaction

- [ ] Interactive elements are keyboard reachable
- [ ] Buttons have clear focus states
- [ ] Circular score buttons are real buttons or accessible radio controls
- [ ] Tap targets are large enough for mobile

## Text and contrast

- [ ] Text contrast is readable in light mode
- [ ] Text contrast is readable in dark mode
- [ ] Important information is not conveyed by colour alone
- [ ] Labels are readable and not too tiny

## Forms

- [ ] Body Scan questions have clear labels
- [ ] Selected states are obvious
- [ ] Error messages are understandable
- [ ] Inputs do not rely only on icons

## Motion

- [ ] Motion is subtle
- [ ] No flashing or distracting animation
- [ ] App remains usable with reduced motion preferences
