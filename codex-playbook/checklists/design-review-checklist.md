# Design Review Checklist

Use this before accepting any Training Arc UI change.

## Brand fit

- [ ] Feels like a cute fitness/lifestyle app
- [ ] Feels warm, soft, practical, and polished
- [ ] Does not feel corporate
- [ ] Does not feel like an admin dashboard
- [ ] Does not feel like a chunky mobile game menu

## Colour

- [ ] Light mode feels strawberry milk / blush / cream / peach
- [ ] Dark mode feels deep rose / warm berry / cocoa / cherry
- [ ] Plum/purple is not the dominant colour
- [ ] Lavender, if present, is only a small accent
- [ ] Dark mode is not muddy brown or cold purple

## Layout

- [ ] Mobile-first layout works well
- [ ] Primary CTA is obvious
- [ ] Cards are rounded and soft
- [ ] Cards are not boxy admin panels
- [ ] Spacing is generous but not wasteful
- [ ] Visual hierarchy is clear

## Components

- [ ] Reuses existing components where possible
- [ ] Buttons are pill-shaped where appropriate
- [ ] Score inputs use circular number buttons
- [ ] No sliders are used
- [ ] Status chips are soft and readable
- [ ] Bottom navigation is subtle and app-like

## Icons

- [ ] Icons are meaningful
- [ ] Icons are not oversized filler
- [ ] Icon style matches soft 3D pastel direction
- [ ] Icons support scanning rather than distracting

## Copy

- [ ] Copy sounds warm and supportive
- [ ] Uses Training Arc terminology correctly
- [ ] Avoids shamey hustle language
- [ ] Avoids corporate wording

## Final decision

- [ ] Accept
- [ ] Needs revision
