# UI Implementation Rules

Training Arc should look like a polished personal lifestyle app.

## Mobile-first

Build mobile first, then expand to desktop.

Mobile screens should feel like real app screens:

- focused top section
- clear primary action
- soft bottom navigation
- generous spacing
- thumb-friendly buttons
- no dense dashboard grids

## Colour rules

Use warm soft colours.

Light mode:

- warm ivory backgrounds
- blush/strawberry primary actions
- peach/soft rose secondary accents
- cocoa text

Dark mode:

- cherry cocoa background
- warm berry cards
- rose/pink highlights
- cream/peach accents

Avoid:

- dominant plum
- cold purple
- muddy brown
- harsh neon
- corporate blue

## Shape rules

Use:

- rounded cards
- pill buttons
- circular score buttons
- soft chips
- gentle dividers

Avoid:

- sharp rectangles
- heavy borders
- big admin panels
- chunky game menu tiles

## Icon rules

Icons should be:

- soft
- small-to-medium
- meaningful
- used to support scanning

Do not use icons as giant filler decorations.

## Inputs

Do not use sliders.

For Energy 1–5, use circular buttons:

- 1 floor
- 2 low battery
- 3 functional
- 4 ready
- 5 main character

For RPE 1–10, use circular buttons:

- 1 chill
- 5 steady
- 10 boss fight

## Motion

Use subtle motion only:

- button press scale
- soft card reveal
- gentle result transition
- tiny success animation

Avoid loud game-like animation.
