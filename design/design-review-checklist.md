# Design Review Checklist

Run this checklist before considering any UI task complete.

## Brand fit

- [ ] Does it feel like a cute fitness companion?
- [ ] Does it feel warm, soft, and practical?
- [ ] Does it avoid corporate/admin dashboard energy?
- [ ] Does it avoid chunky mobile game menu energy?
- [ ] Does it feel polished enough for a personal mobile app?

## Colour

- [ ] Is pink/blush/cream/peach the dominant light mode direction?
- [ ] Is lavender only a small accent?
- [ ] Is plum/purple not dominant?
- [ ] Does dark mode feel deep rose / cherry cocoa / warm berry?
- [ ] Does dark mode avoid muddy brown and cold purple?

## Layout

- [ ] Is the main action obvious?
- [ ] Is the screen mobile-first?
- [ ] Is there enough whitespace?
- [ ] Are there no dense dashboard grids?
- [ ] Are cards soft and rounded rather than boxy?

## Components

- [ ] Are reusable components used consistently?
- [ ] Are buttons pill-shaped or softly rounded?
- [ ] Are score inputs circular buttons?
- [ ] Are there no sliders?
- [ ] Is the theme toggle subtle and pretty?

## Icons

- [ ] Are 3D icons used intentionally?
- [ ] Are icons not oversized or randomly floating?
- [ ] Does the UI avoid looking like stickers pasted onto a dashboard?

## Copy

- [ ] Does the copy use Training Arc terminology?
- [ ] Is the tone supportive and cute without being childish?
- [ ] Are rest/modify states guilt-free?
- [ ] Are result messages practical and clear?

## Accessibility

- [ ] Is text readable?
- [ ] Are tap targets large enough?
- [ ] Is contrast acceptable in both themes?
- [ ] Does the UI support reduced motion?
- [ ] Are status colours not the only indicator of meaning?

## Final pass

- [ ] Would Abby actually want to open this when tired?
- [ ] Does this help answer: “How cooked am I today?”
- [ ] Does this protect the protagonist?
