# Screens

## 1. Home Dashboard

### Goal

Help Abby quickly understand today’s training context and start the Body Scan Quest.

### Main action

“Start Body Scan Quest”

### Required sections

1. Warm greeting
2. Today’s Arc
3. Hero card asking “How cooked are we today?”
4. Training Compass preview
5. Today at a glance widgets
6. Bottom navigation

### Layout

Mobile order:

1. Header: “Morning Abby” / “Good evening, Abby”
2. Today’s Arc card
3. Hero CTA card
4. Training Compass card
5. Tiny widgets grid
6. Bottom nav

### Tiny widgets

- Sleep
- Soreness
- Last workout
- Weekly load
- Mood

### Visual notes

- The hero card should be the main visual focus.
- Use one cute 3D illustration/icon, not many.
- Keep it calm and soft.

## 2. Body Scan Quest

### Goal

Collect enough information to recommend train, modify, rest, or Peace Mode.

### Required inputs

- Energy, 1–5 circular buttons
- Soreness
- Breathing / illness
- Mood / emotional state
- Planned workout
- Recent effort / RPE, 1–10 circular buttons

### Layout

Can be step-by-step or grouped, but must feel quick.

Suggested structure:

1. Header with progress: “Step 1 of 6”
2. Question card
3. Answer controls
4. Back / Next buttons

### Interaction rules

- No sliders.
- Circular buttons for numeric scales.
- Pill chips for options.
- Large tap targets.

## 3. Training Decision Result

### Goal

Give Abby a clear, kind decision and practical next step.

### Possible results

- Slay Day
- Modify the Quest
- Recovery Episode
- Peace Mode

### Required sections

1. Result hero card
2. Why this result happened
3. Recommendation
4. Modified plan, if relevant
5. Tiny win option
6. Save today’s arc button

### Visual notes

- This screen is the emotional payoff.
- Make it feel rewarding even when the result is rest.
- Avoid guilt-inducing language.

## 4. Workout Log

### Goal

Let Abby see saved workouts and body scan results.

### Required sections

1. Header
2. Filter chips: All, Run, Gym, Other
3. This week list
4. Workout cards
5. View all logs action

### Workout card content

- Icon
- Workout title
- Type
- Date
- Duration / distance if relevant
- RPE
- Optional status

### Visual notes

- Should feel like a soft journal, not a data table.

## 5. Weekly Arc View

### Goal

Show the week as a training story.

### Required sections

1. Week strip
2. Daily status bubbles
3. Training load summary
4. Long run / hard day / recovery notes
5. Arc notes
6. Focus this week

### Weekly notes examples

- “Long run protected”
- “Recovery on track”
- “Hard days 2 / 3”
- “Tiny wins count”
- “Recovery debt building”

### Visual notes

- Planner strip, not calendar grid.
- Gentle status colours.
- Small icons for workout type.
