# UI Guidelines

## Layout principles

- Mobile-first at all times.
- Desktop should expand the mobile app layout gracefully.
- Keep screens calm and focused.
- Use one main action per screen.
- Use hierarchy through spacing, size, and soft colour, not heavy borders.
- Keep visual density low enough that the app feels cozy.

## Visual priorities

Each screen should have no more than three visual priorities:

1. Main purpose of the screen.
2. Current recommendation or state.
3. Next action.

Everything else should be secondary.

## Card rules

Cards should be soft, rounded, and layered.

Do:

- use rounded cards
- use warm surfaces
- use soft shadows
- use subtle gradients
- use gentle borders

Do not:

- create boxy admin cards
- use harsh grey borders
- use equal-sized KPI boxes everywhere
- make cards look like a finance dashboard
- overfill cards with icons

## Colour rules

- Pink/rose/peach/cream should define the visual identity.
- Lavender is an accent only.
- Purple must not dominate.
- Dark mode must feel cherry cocoa, not plum.
- Avoid cold greys.

## Controls

Do not use sliders.

Use:

- circular score buttons for Energy and RPE
- pill chips for categorical choices
- segmented controls for filters
- large pill CTA buttons

## Icon usage

Use icons intentionally.

Good icon uses:

- Home hero illustration
- workout type indicator
- sleep/mood/soreness tiny widget
- result state accent
- weekly day bubble

Bad icon uses:

- huge random floating icons
- icons beside every single label
- sticker-like icons pasted into empty corners
- icons that overpower content

## Tone of interface

The UI should feel supportive, not bossy.

Good:

- “Protect the protagonist.”
- “Tiny win accepted.”
- “Still showing up, just smarter.”

Avoid:

- “Failure to meet training target”
- “Low compliance”
- “You are underperforming”
- anything corporate, clinical, or guilt-inducing

## Dark mode rules

Dark mode should be warm and cute.

Use:

- deep cherry cocoa background
- dark rose cards
- warm berry borders
- peach and blush highlights
- subtle glow

Avoid:

- muddy brown
- black/grey admin panels
- cold violet/plum dominance
- neon gamer UI

## Desktop rules

Desktop should not become an admin dashboard.

Use desktop for:

- gentle split layouts
- wider card breathing room
- side preview panels
- soft two-column layouts

Avoid:

- dense grids
- KPI dashboard rows
- table-heavy screens
- analytics-style panels
